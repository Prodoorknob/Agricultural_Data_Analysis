Project Aim

USDA NASS publishes rich agricultural survey data — crop yields, commodity prices, planted acreage, livestock counts, farm labor, economics — but it lives behind a clunky API with no good visual layer on top. Our goal is to make that data explorable.

The dashboard ingests historical NASS data from 2001 to 2025 across all US states, normalizes it into a clean data model, and presents it through a narrative-driven interface that answers agricultural questions — not just dumps numbers. Think of it as a Google Analytics for US farming data: fast, filterable, and built around what someone actually wants to know.

What the Dashboard Does
Page 01
Overview Dashboard
High-level snapshot of US agriculture. State selector, key KPIs, area trends over time, and crop progress bar chart.
  • State-level filter with cross-page sync
  • Area Trends chart (planted vs. harvested)
  • Top crops by fastest growth
  • Crop progress completion %
Page 02
Crops Deep-Dive
Narrative-driven crop analysis. Follows a causal story: yield → acreage → revenue. Anomaly detection surfaced inline.
  • KPI cards with YoY % + sparklines
  • Yield dip detection (e.g. 2002, 2012)
  • Harvested vs. planted discrepancy flags
  • Commodity switcher (corn, soy, wheat…)
Page 03
Economics & Prices
Prices received by farmers over time. Connects market price trends to production context.
  • Price time series by commodity
  • Revenue trend over 20+ years
  • Year-over-year price change view
Page 04
Livestock
Animal agriculture counts and values. Head counts, pricing, and production breakdowns by species.
  • Livestock inventory by type
  • Price received trends
  • State-level filtering
Page 05
Labor
Farm workforce data — wage rates, worker counts, and labor composition over time.
  • Hired vs. family labor split
  • Regional wage comparisons
  • Seasonal labor trends
Tech Stack
🖥 Frontend
Next.js
React framework powering all dashboard pages. Handles routing, server-side rendering, and API calls.
React
Component library for all UI elements — charts, KPI cards, filters, tables.
Recharts / Chart.js
Charting libraries for time series, bar charts, sparklines, and trend visuals.
Tailwind CSS
Utility-first styling system keeping the UI consistent and fast to iterate.
⚙️ Backend
Python
Core language for the data pipeline — API ingestion, transformation, validation, and loading.
FastAPI
REST API layer that the Next.js frontend queries. Serves pre-processed data to dashboard endpoints.
Pandas / NumPy
Data wrangling — normalization, reshaping, anomaly flagging, derived metric computation.
PostgreSQL
Relational database storing the cleaned, partitioned NASS data. Indexed by state, commodity, and year.
🔄 Data Pipeline
USDA QuickStats API
Source of all agricultural data. Queried with parameterized requests by category, state, and year range.
ETL Scripts
Python scripts handling Extract → Transform → Load. Handles rate limits, pagination, and schema validation.
Scheduled Jobs
Automated pipeline runs to keep data current as NASS publishes new survey results.
☁️ Infrastructure
AWS EC2
Ubuntu 24.04 server hosting the FastAPI backend and database. Accessed via SSH key pairs.
AWS IAM / SSM
Access control and secure instance management. IAM roles configured for pipeline permissions.
Vercel / GitHub
Next.js frontend deployed on Vercel. Source code version-controlled on GitHub.
