1. What This Module Does

The Crop Yield Prediction module estimates bushels-per-acre output for a given county before harvest occurs — issuing county-level forecasts for corn, soybeans, and wheat as the growing season progresses. A forecast generated in July reads: "Iowa, Story County — corn projected at 192 bu/acre (±11%), down from last year's 198."

The model integrates three signal families: weather stress (temperature anomalies, cumulative precipitation deficit, drought severity), soil suitability (drainage class, plant-available water capacity from SSURGO), and crop condition ratings (weekly USDA NASS 5-tier condition scores as a leading indicator). Forecast confidence widens early in the season and tightens materially by late July.

Honest caveat: A naive county yield average frequently outperforms ML models when historical data is sparse. Early-season forecasts (April–May) should be presented with wide confidence intervals. The model's true value emerges July–August when in-season signals are rich and the baseline has little additional signal left to give.

All data sources are free government APIs — no licensing fees. Several are already present in the existing USDA NASS pipeline; the remainder require lightweight one-time setup.

Key Feature Engineering Decisions

Growing Degree Days (GDD): Accumulated heat units from planting to current week; computed from NOAA daily temps using base temperature of 50°F for corn. GDD-to-date is the single strongest in-season predictor. Crop Condition Index (CCI): NASS weekly ratings collapsed to a numeric score (Good/Excellent = +2, Fair = 0, Poor/Very Poor = −2), cumulated over the season. Soil Adjusted Precipitation: Cumulative rainfall weighted by SSURGO available water capacity — high-capacity soils buffer drought stress, low-capacity soils amplify it.

3. Model Implementation

Recommended Architecture: Gradient Boosting with Temporal Cross-Validation

XGBoost or LightGBM is the right primary choice for this task — not deep learning. The dataset is tabular, features are well-defined, and the signal-to-noise ratio demands interpretability. Crop yield ML literature consistently shows gradient boosting matching or exceeding LSTM on structured feature sets at county granularity.

1
Data Ingestion & Temporal Join
Pull weekly snapshots from NASS + NOAA + Drought Monitor. Join on FIPS county code + crop year + week-of-season. SSURGO soil features are a static left join — pulled once, merged by FIPS.
pandas merge on [fips, year, week] → feature matrix
2
Feature Construction
Compute GDD accumulated to current week, CCI rolling sum, cumulative precip deficit vs. 30-year normal, VPD stress days (days exceeding 2.0 kPa). All features indexed as "week W of season" to make them comparable across calendar years.
features: [gdd_ytd, cci_cumul, precip_deficit, vpd_stress_days, drought_d3d4_pct, soil_awc, soil_drain_class]
3
Training Strategy: Walk-Forward Validation
Never train on future years. Use leave-future-out cross-validation — train on years 2000–2018, validate on 2019–2021, test on 2022–2024. Prevents data leakage that inflates accuracy metrics. Retrain annually as new season closes.
TimeSeriesSplit(n_splits=5, gap=1_year)
4
Model Fitting: Per-Crop, Per-Week
Train a separate model per crop × week-of-season combination. A week 20 corn model uses only signals available through week 20; a week 30 model gets the fuller picture. This means ~3 crops × 20 weekly snapshots = 60 models, but each is lightweight and interpretable.
XGBRegressor(n_estimators=400, max_depth=5, learning_rate=0.05)
5
Uncertainty Quantification
Use quantile regression (LightGBM native support) to produce 10th–90th percentile forecast bands, not just point estimates. The UI should always display the confidence interval — particularly critical for early-season forecasts where uncertainty is high.
LGBMRegressor(objective='quantile', alpha=0.1 / 0.5 / 0.9)
6
Forecast Storage
Write predictions to PostgreSQL (existing pipeline DB). Table: yield_forecasts(fips, crop, week, year, p10, p50, p90, model_version). Immutable — never overwrite historical forecasts so post-season accuracy auditing is preserved.

Frontend Components

County choropleth overlay: Color-coded by forecast deviation from county historical average (z-score). Green = above average, red = below average. Existing map component extended with a "Yield Forecast" layer toggle. Forecast confidence strip chart: A simple visualization showing how forecast p10–p90 bands narrow as the season progresses — communicates model maturity without technical jargon. USDA Benchmark callout: Each county forecast card displays the most recent NASS published actual alongside the current model forecast for easy comparison.

Weekly Automation

A Python script runs every Thursday (triggered via cron on existing AWS EC2 instance) after NASS publishes new crop condition data. Runtime is approximately 8–12 minutes for all county × crop combinations. Failures alert via existing monitoring stack; the previous week's forecast remains live until a new run succeeds.

5. Risks & Mitigations

High
Early-season overconfidence in UI. Users may treat April forecasts as reliable. Mitigation: Force-display confidence interval bands in UI; dynamically label forecast confidence tier ("Low — early season", "High — post-pollination") based on week of season.
High
NOAA station data gaps for sparse counties. Rural counties may have no nearby weather station with complete records. Mitigation: Fall back to gridded reanalysis data (PRISM or NOAA GRIDMET) for counties where nearest station has >10% missing values.
Medium
Walk-forward validation understates generalization error. Training on 2000–2018 may miss structural shifts (climate change, new hybrids). Mitigation: Recalibrate annually; include a year-trend feature to capture systemic yield improvement over time.
Low
SSURGO data staleness. Soil data was compiled over multiple decades; recent drainage improvements may not be reflected. Mitigation: Document vintage in data dictionary; flag for periodic manual review.
