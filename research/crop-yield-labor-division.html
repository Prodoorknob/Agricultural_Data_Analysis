Cross-domain work happens at integration points (API contract, DB schema), which are jointly agreed before work begins.

4. Team Coordination Protocol

Weekly Sync (30 min)

  • Each member demos what they completed vs. last week's gate
  • Blockers raised — Lead resolves infra blockers same-day
  • Next week's tasks assigned and confirmed
  • No async updates substitute for this meeting during crunch weeks

GitHub Workflow

  • Branch per task: feature/nass-ingest, feature/gdd-engine
  • PRs require at least one review before merge to main
  • CI must pass (lint + unit tests) before merge
  • Lead reviews all PRs touching the DB schema or API layer

Shared Resources

  • AWS credentials: Lead manages IAM; teammates use named IAM users
  • Secrets: .env shared via encrypted channel; never committed to git
  • S3 bucket: all members have read access; write access scoped per role
  • RDS: separate app user (ag_app) with minimal privileges

Escalation & Blockers

  • If a task is blocked > 24 hours, post in team channel immediately
  • Lead is the single point of escalation for AWS/infra issues
  • ML lead is the single point of escalation for data quality issues
  • Schedule scope: if a feature risks the timeline, cut it — don't delay the demo

5. Team-Level Risks

High
Beginner contributor blocked on SSURGO parsing. SSURGO bulk download format is a non-obvious GIS-adjacent format (shapefile + tabular). Mitigation: Lead pre-downloads and unzips the SSURGO file in Week 1; provides Contributor with a starter script that reads the right table (muaggatt) and just needs the FIPS join logic added.
High
API contract disagreement between Lead and ML Lead mid-project. If the feature matrix schema changes after the API contract is signed off, both modules need updates. Mitigation: Week 3 API contract review is a hard gate — both leads must sign off. Any schema change after that requires a joint decision, not unilateral commits.
Medium
ML Lead underestimates walk-forward validation time. Training 60 models plus proper hold-out CV on a t3.medium can take longer than expected if the historical dataset is large. Mitigation: Start training in Week 4 not Week 5; spot-check on a 5-state subset first to validate training pipeline before full run.
Low
AWS cost overrun. If EC2 is left running continuously, cost exceeds budget. Mitigation: Lead sets a CloudWatch billing alarm at $40/month; EC2 is stopped manually after each work session until the EventBridge cron is live.
