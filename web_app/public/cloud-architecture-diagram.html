Design bets this diagram encodes. (1) Training stays off EC2 — local PC or Colab only — which keeps the serving instance at t3a.small. (2) Models are pickles in S3 with HMAC signatures; the serving process re-hydrates on restart, so EC2 is effectively stateless at the model layer. (3) Dual S3 layout (partitioned + Athena-optimized) is the price of both fast browser reads and ad-hoc SQL — compression savings from a single layout weren't worth the lost capability. (4) The Publicly Accessible = Yes posture on RDS is intentional for dev ergonomics but flagged for VPC migration before any public demo. (5) Cron + systemd instead of Lambda / ECS — pandas + pyarrow + yfinance together bust Lambda's 250 MB deployment limit and the cold-start penalty on bulk ETL would dwarf any savings. See the full report for rationale on every omission.
